# Phenoscribe vs Control coding on a long-COVID email

## Headline

Two HPO code lists were prepared for the same email — one by Phenoscribe, one by hand (the **Control**). Side by side:

| | Control | Phenoscribe |
|---|---|---|
| HPO codes proposed | 8 | 15 rows / 8 distinct concepts |
| IDs whose stated label matches what HPO actually calls that ID | 2 / 8 | 14 / 15 |
| Concepts from the email that are captured | 8 / 8 | 7 / 8 |
| Codes that don't fit the source quote | 6 | 1 |

Six of the eight Control IDs point to HPO terms that are different from the label written next to them — including a pelvic-bone shape and a bleeding-tendency term. Phenoscribe's 15 rows agree with their own labels in 14 cases, cover seven of the eight Control concepts, and add one genuine misfire (HP:0033750) worth tracking.

## What the two lists are

- **Phenoscribe** — the result of running the pipeline on the email.
- **Control** — a manually-prepared HPO list sent alongside the email as a comparison reference.

## The email

A long-COVID patient writes to their disability evaluator. The text is a 4374-character first-person account, in French, explaining that missing an administrative instruction reflects underlying cognitive and fatigue limitations. Symptoms are stated twice: once in detail (paragraphs 5–6), once as a closing summary (paragraph 7).

## Control list, checked against HPO

Each Control ID was looked up in the current HPO release.

| Control ID | Label in the Control | What HPO calls that ID | Match? |
|---|---|---|---|
| HP:0000739 | Trouble des fonctions exécutives | Anxiety | ✗ |
| HP:0031026 | Difficulté d'organisation | Snail-like ilia (a pelvic-bone shape) | ✗ |
| HP:0000736 | Difficulté de concentration | Short attention span (paediatric ADHD term) | ✗ |
| HP:0012378 | Fatigue | Fatigue | ✓ |
| HP:0003546 | Intolérance à l'effort | Exercise intolerance | ✓ |
| HP:0000978 | Réduction de l'endurance physique | Bruising susceptibility | ✗ |
| HP:0012531 | Stress psychologique | Pain | ✗ |
| HP:0033676 | Épuisement | Posttraumatic stress symptom | ✗ |

Only Fatigue and Exercise intolerance carry an ID that actually resolves to the term written next to it. The six others are clinically reasonable *labels* for the email's content — the ID strings just point elsewhere in the ontology. This is the recurring pattern with HPO codes assigned by hand or by an LLM without ontology grounding: the names land, the identifiers slip.

## Phenoscribe list, reconciled to the Control's concepts

Phenoscribe produced 15 rows. Mapped back to the eight concepts the Control was trying to capture:

| Concept the Control was capturing | Phenoscribe code | Notes |
|---|---|---|
| Difficulty interpreting administrative instructions | HP:0033051 Impaired executive functioning | Correct ID for the concept. |
| Organising documents from multiple sources | HP:0033056 Impaired ability to organize | Correct ID for the concept. |
| Concentration / focus problems | HP:0031987 Diminished ability to concentrate | The adult-coded attention term. |
| Effort of one appointment exhausts the day | HP:0012378 Fatigue | Same ID as the Control. |
| One important task mobilises all energy | HP:0003546 Exercise intolerance | Same ID as the Control. |
| Cannot chain activities across a day | HP:0030973 Postexertional symptom exacerbation | Canonical long-COVID PEM term. |
| Stress level generated by procedures | HP:0000739 Anxiety | HPO has no clean "psychological stress" term; Anxiety is the closest. |
| "J'étais épuisé" | folded into HP:0012378 Fatigue | No separate row for exhaustion. |

## What Phenoscribe adds beyond the Control

- **HP:0100543 Cognitive impairment** — appears twice, as a summary code over the cognitive cluster.
- **HP:0012432 Chronic fatigue** — pulled from "problèmes de fatigue" in the closing summary; arguably more specific than plain Fatigue given long-COVID context.
- Several concepts (executive function, organisation, concentration, cognitive impairment) appear 2–3 times because the email restates the same symptoms in its closing paragraph. The extractor preserves each quote as its own row. For a batch of clinical interviews this is the right behaviour. For a single short email it makes the output noisier than it needs to be.

## The one Phenoscribe misfire

**HP:0033750 — Reduced functional residual capacity** for the quote *"incapacité à maintenir un niveau de fonctionnement constant sur la durée"*.

HP:0033750 is a lung-volume measurement. The patient's quote is about cognitive and functional endurance. The candidate shortlist for that line surfaced respiratory terms (likely because words like "functional" and "capacity" sit close to lung terminology in the embedding space), and the final pick landed on the least-bad of five wrong options. Worth keeping as a regression case.

## Takeaways

1. **The Control's labels are clinically sensible; the IDs sitting next to them often are not.** Anyone reading the Control by label alone would see a fair summary of the email. Anyone using the IDs downstream — to query, aggregate, or feed into another system — would pull bone-shape and bleeding-tendency diagnoses that the patient never mentioned. This is precisely the problem Phenoscribe was built to solve.
2. **Phenoscribe is internally consistent: each ID resolves to the term written next to it.** It covers seven of the Control's eight concepts and adds one (Postexertional symptom exacerbation) that fits long-COVID well.
3. **Phenoscribe has work left.** One misfire (HP:0033750), no distinct row for "épuisement", and visible duplication when the source restates symptoms. Both are tractable.

## Suggested follow-ups

- Track HP:0033750 as a regression case so future tuning has to keep it correct.
- Add a dedup pass for single-document outputs that keeps one row per (code, patient) with the most-specific quote.
- Run the same comparison on a second email or transcript to see whether the pattern (Control labels fine, Control IDs partially off; Phenoscribe IDs internally consistent) repeats.
